Getting Started

EduBots is a web-based block programming platform designed for educational robotics. Follow these steps to get started:

1. Installation

You can run EduBots using Docker or as a standalone application:

Using Docker (Recommended)

# Pull the Docker image
docker pull openroberta/lab

# Run the container
docker run -d --name edubot -p 1999:1999 openroberta/lab

# Access at http://localhost:1999

Local Installation

See the Download page for installation packages.

2. Connecting Your Robot

To program your robot, you need to:

  1. Connect your robot to your computer via USB
  2. Download and run the OpenRoberta Connector
  3. In the web interface, click the "Connect" button
  4. Select your robot type from the dropdown menu
OpenRoberta Connector: This is a Java application that enables communication between the web browser and your USB-connected robot. Download it from the Download page.

3. Creating Your First Program

  1. Select your robot from the robot menu (e.g., NanoDiffBot, mBot, Arduino Nano)
  2. Drag blocks from the toolbox on the left
  3. Connect blocks to create your program
  4. Click "Run" to compile and upload to your robot

Supported Robots

NanoDiffBot

Arduino Nano-based differential drive robot with the following features:

  • Motors: L298N motor driver controlling two DC motors
  • Sensors: Ultrasonic distance sensor, light sensor, line following sensors
  • Actuators: RGB LED, buzzer
  • Input: IR receiver for remote control

NanoDiffBot Pin Configuration

Left Motor:  PWM=5, DIR1=7, DIR2=8
Right Motor: PWM=11, DIR1=12, DIR2=A4
Ultrasonic:  TRIG=4, ECHO=2
Light Sensor: A0
Line Sensors: LEFT=A1, RIGHT=A2
Buzzer: Pin 9
IR Receiver: Pin 10
RGB LED: RED=3, GREEN=6, BLUE=A3

Arduino Boards

Direct support for programming Arduino Nano, Uno, and Mega boards with standard Arduino blocks.

mBot & mBot2

Makeblock educational robots with built-in sensors, motors, and MicroPython support (mBot2/CyberPi uses ESP32).

Troubleshooting

Robot Not Detected

Problem: Robot is not showing up in the connection dialog

Solutions:

  • Make sure the OpenRoberta Connector is running
  • Check USB cable connection
  • Install USB drivers for your robot (CH340 for Arduino Nano clones)
  • Try a different USB port or cable
  • Check Device Manager (Windows) or ls /dev/tty* (Mac/Linux) to verify the device is detected

Compilation Errors

Problem: Program won't compile or upload

Solutions:

  • Check that you selected the correct robot type
  • Verify all blocks are properly connected (no red error indicators)
  • Make sure the robot is not running another program (press reset button)
  • Check the console for specific error messages

Docker Issues

Problem: Docker container won't start or is slow

Solutions:

  • On Apple Silicon Macs: The AMD64 image runs under emulation, which may be slower
  • Allocate more resources to Docker (Settings â†’ Resources)
  • Check Docker logs: docker logs edubot
  • Ensure port 1999 is not in use by another application

Block Programming Basics

Control Blocks

  • Start: Every program begins with a start block
  • Loop: Repeat actions multiple times or forever
  • If/Else: Make decisions based on conditions
  • Wait: Pause program execution

Motor Blocks

  • Drive: Move forward or backward at a specific speed
  • Turn: Rotate left or right
  • Stop: Stop all motors
  • Steer: Control left and right motors independently

Sensor Blocks

  • Ultrasonic Distance: Measure distance in centimeters
  • Light Sensor: Detect light levels
  • Line Sensor: Detect black/white lines for line following

Output Blocks

  • LED: Control RGB LED colors
  • Buzzer: Play tones and melodies

Example Programs

Move Forward

Simple program to move the robot forward for 2 seconds:

  1. Start block
  2. Drive forward at speed 50
  3. Wait 2000ms (2 seconds)
  4. Stop motors

Obstacle Avoidance

Robot stops when it detects an obstacle closer than 20cm:

  1. Start block
  2. Forever loop
  3. If ultrasonic distance < 20cm
  4. Then: Stop motors and wait
  5. Else: Drive forward at speed 50

Line Following

Follow a black line on white surface:

  1. Start block
  2. Forever loop
  3. If both sensors on line: Drive forward
  4. If left sensor on line: Turn left
  5. If right sensor on line: Turn right
